Global settings define the default values used by every PDF feed in Fluent PDF. Individual feeds can override any of these values — the global settings act as your site-wide baseline.

## Where to find global settings

Global settings are accessible from two places:

- **Fluent Forms → Add-Ons → Fluent PDF** — the add-on page within Fluent Forms (Settings tab)
- **Fluent PDF → Global Settings** — the standalone Fluent PDF top-level admin menu

<Note>
  Settings are stored in the WordPress options table under the key `_fluent_pdf_settings`.
</Note>

## Settings reference

<ParamField path="paper_size" type="string" default="A4">
  The default paper size for all generated PDFs. Accepts any size key from the [full list of supported sizes](/configuration/paper-and-layout).

  Common values: `A4`, `Letter`, `Legal`, `A3`, `A5`.
</ParamField>

<ParamField path="orientation" type="string" default="P">
  The default page orientation.

  | Value | Label |
  |-------|-------|
  | `P` | Portrait |
  | `L` | Landscape |
</ParamField>

<ParamField path="font_family" type="string" default="default">
  The default font used for body text. The dropdown is grouped by script category (Unicode, Indic, Arabic, CJK, and others). Available options come from the installed font pack — see [Font management](/configuration/fonts) to learn how to download fonts.

  The field uses a `dropdown-group` component, so options are organized under headings matching each script category.
</ParamField>

<ParamField path="font_size" type="number" default="14">
  The default body font size in points. Must be a positive integer. The value is cast to an integer before being saved.
</ParamField>

<ParamField path="font_color" type="string" default="#323232">
  The default color for body text. Accepts any valid hex color code. Set using the color picker in the admin UI.
</ParamField>

<ParamField path="heading_color" type="string" default="#000000">
  The default color for headings. Accepts any valid hex color code.
</ParamField>

<ParamField path="accent_color" type="string" default="#989797">
  The default accent color. Used for borders, horizontal rules, and other decorative breaks in the PDF layout. Accepts any valid hex color code.
</ParamField>

<ParamField path="language_direction" type="string" default="ltr">
  The text direction for rendered content.

  | Value | Label | Use case |
  |-------|-------|----------|
  | `ltr` | LTR | Left-to-right scripts (Latin, Cyrillic, etc.) |
  | `rtl` | RTL | Right-to-left scripts (Arabic, Hebrew, etc.) |

  For Arabic or Hebrew content, select **RTL** and use a compatible font from the Arabic font group. See [Paper size and layout](/configuration/paper-and-layout) for more on language direction.
</ParamField>

## Per-feed overrides

Every setting on this page is a default. When you configure an individual PDF feed under **Fluent Forms → PDF Feeds**, the feed-level settings take precedence over these globals. If a feed setting is left blank, the global value is used as the fallback.

<Tip>
  Set your most common paper size, font, and colors here so individual feeds only need to specify what's different.
</Tip>
