This guide walks you through creating a PDF feed on a Fluent Form, submitting a test entry, and downloading the resulting PDF. By the end you will have a working PDF feed and know how to embed a download link in a confirmation message.

<Note>
  Complete [installation](/installation) before starting this guide. Fluent Forms must be installed and Fluent PDF fonts must be downloaded.
</Note>

## Create and test your first PDF feed

<Steps>
  <Step title="Open a form in Fluent Forms">
    In your WordPress admin, go to **Fluent Forms** and open any existing form. If you don't have one yet, create a simple contact form with a few fields.
  </Step>

  <Step title="Go to PDF Feeds">
    Inside the form editor, click **Settings** in the top navigation, then select **PDF Feeds** from the settings menu.

    You will see an empty feed list. Each form can have multiple feeds — for example, one for an internal record and another formatted as a receipt.
  </Step>

  <Step title="Create a new feed">
    Click **New Feed**. Fluent PDF will show you the available templates:

    - **General** — a flexible layout for any form entry, with configurable header, body, and footer sections.
    - **Invoice** — a tabular payment receipt (only shown for forms with a payment field).

    Select **General** to continue.
  </Step>

  <Step title="Configure the feed content">
    The feed editor has two sections:

    **Content** — controls what appears in the PDF body. The body field accepts HTML and Fluent Forms shortcodes. The default body pre-populates with `{all_data}` to output all form fields. You can also set a custom header and footer that appear on every page.

    **Appearance** — overrides the global defaults (paper size, orientation, font, colors, watermark) for this specific feed.

    For this first feed, leave the defaults in place and give the feed a name like `Submission Summary`.
  </Step>

  <Step title="Enable the download link and save">
    Scroll to the **Allow Download** option and set it to **Yes**. This makes the feed available as a downloadable link for logged-in users.

    Click **Save** to create the feed. After saving, a **Shortcode** field appears showing the feed-specific shortcode:

    ```
    {pdf.download_link.42}
    ```

    (The number is your actual feed ID.) Copy this shortcode — you will use it in the next step.
  </Step>

  <Step title="Add a download link to the confirmation message">
    Go to the form's **Confirmation** settings and paste the shortcode into the confirmation message. For example:

    ```
    Thank you for your submission! Download your PDF here: {pdf.download_link.42}
    ```

    You can also use the generic shortcode `{pdf.download_link}` to link to the first feed on the form, without specifying a feed ID.

    Save the confirmation settings.

    <Tip>
      The same shortcodes work in Fluent Forms notification emails, so you can send PDF links to both the submitter and your team.
    </Tip>
  </Step>

  <Step title="Submit a test entry">
    Preview the form and fill it out with test data. Submit the entry.

    The confirmation message should appear with a rendered download link in place of the shortcode.
  </Step>

  <Step title="Download the PDF">
    Click the download link. Fluent PDF will generate the PDF and stream it to your browser.

    You can also download PDFs for any entry from the Fluent Forms submission viewer: open **Fluent Forms → Entries**, open a submission, and look for the **PDF Downloads** widget in the right sidebar.
  </Step>
</Steps>

## Shortcode reference

Two shortcode formats are available in confirmation messages, notification emails, and anywhere Fluent Forms parses shortcodes:

| Shortcode | Description |
|---|---|
| `{pdf.download_link}` | Download link for the first PDF feed on the form |
| `{pdf.download_link.{feedId}}` | Download link for a specific feed by its numeric ID |

Replace `{feedId}` with the actual ID shown in the feed editor's **Shortcode** field.

<Note>
  Download links require the user to be logged in unless the feed uses the public download option. See [Shortcodes](/fluent-forms/shortcodes) for details on public vs. authenticated links.
</Note>

## Generate a PDF programmatically

For developers who need to generate a PDF outside of Fluent Forms — from a custom plugin, theme, or admin page — Fluent PDF exposes a `do_action` hook that accepts an array of HTML strings:

```php
do_action('fluent_pdf_make', [
    'header' => '<h1>Invoice #1234</h1>',
    'footer' => '<p>Thank you for your business.</p>',
    'body'   => '<p>Order summary goes here.</p>',
]);
```

Call this action to stream a PDF directly to the browser. Use `die()` immediately after to prevent any further output:

```php
if (isset($_GET['download_my_pdf'])) {
    do_action('fluent_pdf_make', [
        'header' => '<h1>My Document</h1>',
        'body'   => '<p>Content here.</p>',
        'footer' => '<p>Page footer.</p>',
    ]);
    die();
}
```

See [Generating PDFs programmatically](/developer/generating-pdfs-programmatically) for a full reference including how to pass appearance settings and custom filenames.
