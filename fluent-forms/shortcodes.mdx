---
title: "PDF shortcodes"
description: "Embed PDF download links in form confirmation messages and notification emails using Fluent PDF shortcodes."
---

Fluent PDF adds shortcodes to the Fluent Forms shortcode picker so you can embed PDF download links anywhere Fluent Forms smart codes are supported — including form confirmation messages and notification email bodies.

## Available shortcodes

### `{pdf.download_link}`

Links to the PDF generated by the **first** PDF feed attached to the form.

```
{pdf.download_link}
```

- Resolves to a private AJAX download URL.
- Requires `allow_download` to be set to **Yes** in the feed settings.
- Only works for logged-in users.

### `{pdf.download_link.{feedId}}`

Links to the PDF generated by a **specific feed**, identified by its feed ID.

```
{pdf.download_link.42}
```

- Resolves to a private AJAX download URL for feed `42`.
- Requires `allow_download` to be set to **Yes** in that feed's settings.
- Only works for logged-in users.

<Tip>
  You don't need to look up the feed ID manually. Each feed's settings page displays a read-only **Shortcode** field containing the correct `{pdf.download_link.{feedId}}` value for that feed. Copy it from there.
</Tip>

### `{pdf.download_link.public.{feedId}}`

Links to a publicly accessible PDF download for a specific feed — no login required.

```
{pdf.download_link.public.42}
```

- Resolves to an encrypted public URL.
- The submission ID and feed ID are encrypted using `Protector::encrypt()` and base64-encoded before being included in the URL, so the actual IDs are never exposed.
- Accessible to anyone who has the link, including unauthenticated visitors.

<Note>
  This shortcode is not listed in the Fluent Forms shortcode picker UI (only the private variants appear there). Use it manually by typing the shortcode with your feed ID into the confirmation message or email body.
</Note>

## Where shortcodes work

| Location | Works |
|---|---|
| Form confirmation message | Yes |
| Notification email body | Yes |
| Other email fields (subject, from name, etc.) | Not recommended — shortcodes are parsed as plain text in those fields |
| WordPress page/post content | No — these shortcodes only resolve in Fluent Forms contexts |

## Authentication requirements

<AccordionGroup>
  <Accordion title="Private links ({pdf.download_link} and {pdf.download_link.{feedId}})">
    The download AJAX endpoint (`fluentform_pdf_download`) requires:

    1. A valid `fluent_forms_admin_nonce` in the request.
    2. The user must be logged in.
    3. The user must either have the `fluentform_entries_viewer` capability **or** be the submitter of the entry (matched by `user_id` on the submission record).

    If the user is not logged in or does not have permission, the download returns a `422` error.
  </Accordion>
  <Accordion title="Public links ({pdf.download_link.public.{feedId}})">
    The public download AJAX endpoint (`fluentform_pdf_download_public`) is registered with `wp_ajax_nopriv_`, so it is accessible without logging in.

    The submission ID and feed ID in the URL are encrypted and base64-encoded. The server decrypts them using `Protector::decrypt()` before fetching the submission and generating the PDF. Guessing or tampering with the URL parameters will produce an invalid ID and the request will fail.

    No nonce is required for public links.
  </Accordion>
</AccordionGroup>

## AJAX endpoints

| Endpoint action | Access | Description |
|---|---|---|
| `fluentform_pdf_download` | Logged-in users with permission | Downloads the PDF for a private link. |
| `fluentform_pdf_download_public` | Public (no login required) | Downloads the PDF for an encrypted public link. |

## Example: confirmation message with a download link

The following confirmation message includes both a thank-you note and a link to download the submission PDF:

```
Thank you for your submission, {inputs.first_name}!

You can download your PDF receipt here: {pdf.download_link.42}
```

Replace `42` with the feed ID shown in the **Shortcode** field of your PDF feed settings.

For a public link that works even for guest submitters:

```
Thank you for your order!

Download your invoice: {pdf.download_link.public.42}
```
